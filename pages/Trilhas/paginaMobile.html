Trilha Mobile

Fundamentos

Os fundamentos no contexto de desenvolvimento mobile referem-se aos conceitos e habilidades básicas que um desenvolvedor precisa adquirir antes de avançar para tópicos mais complexos.

Introdução à Programação:

  • Conceitos básicos de lógica de programação.
  • Estruturas de controle (condicionais, loops).

Aprendendo uma Linguagem de Programação:

  • Escolha uma linguagem de programação para desenvolvimento mobile (ex: Kotlin ou Java para Android, Swift ou Objective-C para iOS).
  • Sintaxe básica e estruturas de dados.

Desenvolvimento para Plataformas Móveis:

  • Compreensão da arquitetura de aplicativos móveis.
  • Ambientes de desenvolvimento (Android Studio para Android, Xcode para iOS).
  • Interfaces de usuário básicas.

Intermediário

O segundo módulo aprofunda os conhecimentos, permitindo que os alunos construam aplicações mais complexas e integradas.

Desenvolvimento de Aplicações Nativas:

  • Aprofundamento nas APIs nativas das plataformas (Android SDK, iOS SDK).
  • Manipulação de componentes específicos de cada plataforma.

Arquitetura de Software:

  • Padrões de arquitetura de software (MVC, MVVM).
  • Separação de preocupações.

Persistência de Dados Avançada:

  • Utilização de ORM (Object-Relational Mapping) ou ferramentas similares.
  • Estratégias de backup e segurança.

Testes de Software:

  • Testes unitários para lógica de negócios.
  • Testes de interface do usuário.

Avançado

O terceiro módulo prepara você para assumir projetos completos, compreendendo a arquitetura e a segurança de ponta a ponta.

Desenvolvimento Multiplataforma:

  • Frameworks como Flutter ou React Native.
  • Compartilhamento de código entre plataformas.

Otimização de Desempenho:

  • Perfis de desempenho e otimização de código.
  • Trabalho com imagens e recursos pesados.

Segurança e Privacidade:

  • Boas práticas de segurança em aplicativos móveis
  • Gerenciamento de dados sensíveis e autenticação segura.

Arquiteturas Avançadas:

  • Padrões arquiteturais modernos (Clean Architecture, Redux).
  • Injeção de dependência e inversão de controle.
